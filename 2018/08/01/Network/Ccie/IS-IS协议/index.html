CCIE

1：IS-IS中间系统到中间系统

IGPS阵营中的协议

两大阵营：EGPS和IGPS

IGPS 内部网关协议集

IGPS：RIP OSPF EIGRP IS-IS 都是AS内的协议

EGPS：BGP AS间、AS内

IS-IS正在取代OSPF

学习RIP, EIGRP是为了学习OSPF和BGP？

Link State 链路状态路由协议

根据协议自身特点分类：【分类记住路由协议】

距离矢量 RIP 高级距离矢量 EIGRP 不是靠跳数 公式【基于带宽延迟】+Metric值 但是受距离限制 OSPF 链路状态

*为什么要使用IS-IS取代OSPF？

2：IS-IS的发展历程

同时进行的，没有谁早谁晚的说法。

OSI七层模型 传输层 面向链接TCP 无链接UDP

OSI中后期产物 计算机网络最早的发展阶段提出的是CLNS和CONS

CLNS和CONS算是OSI的前身

N代表网络，S代表服务，C代表链接 O代表Of“有链接的网络服务”类似于TCP L代表Less “无链接的网络服务”

基于对CONS的研究，延伸出了IP【网络标识】和TCP

TCP数据传输控制 IP网络标识

因此开发出了动态的路由协议：RIP EIGRP OSPF BGP

另一阵营：CLNS坚持使用无链接的网络协议

开发出了CLNP：CLNS网络标识

还需要的机制：CLNS【名字重复】IS-IS

OSPF为IP服务，IS-IS为CLNP网络服务。

这些都是不同组织在同时发展的，最后CONS阵营的方案得到公司的认可。

最后，ISO组织提出了OSI七层模型，但是CLNS的理念也得到认可，因此保留下来，叫做UDP

最后，IS-IS就感觉要消失了

这是为了统一行业标准。

OSPF虚链路：①克服孤岛区域：0，1，2区域不互相学路由

②两边区域0，中间区域1，但是两边区域0不互相学路由。

IS-IS没有这个问题。

外企推运营商用IS-IS 思科排第一

割接：干掉原有协议，换新协议

两种方法：①升级OSPF，使其拥有IS-IS特性

②直接用IS-IS取代OSPF

两种方法都不太科学……

所以我们现在使用的IS-IS，并不是纯IS-IS

集成IS-IS、双栈IS-IS

所以很多技术原来与IP对立，后来被IP吞并……

3：CLNP标识

3.1 NET 网络实体标识

类似于IP+MAC组合

3.2 NSAP 网络服务接入点

实现IP+MAC+端口号port_nam

CLNP觉得没有必要把IP+MAC分开。

NSAP在这个基础上把端口号加入。

一步到位。地址就会很长。

3.3 CLNP标识格式

AFI A地址 F格式 I授权组织

数字由官方定好。47，48，49是用在局域网或私网的地址

所以官方有一张表。

0000 完整的一段，但有两块信息。前两个00代表域标识，后两个00代表区域标识

区域被域包含 域就是AS

巨型网络有不同的域标识

0001 0002 0003 同一域中的不同区域

sys-ID——MAC地址

所以说CLNP地址类似于IP+MAC，保证了全球唯一独立性

最后又有两个00 相当于端口号

CLNP全都是16进制 0A代表端口10

地址格式和MAC地址一样，但不是MAC地址

sys-id只要不一样就好，建议使用设备MAC地址……

*如何区分是NET还是NSAP？

看最后的部分是不是00

IS-IS启用的时候，必须要有一个NET地址。

其实是因为IS-IS被同化，留着NET已经没啥用了……

就用NET当ROUTE ID RID

类似于奥运会上的中华台北

3.4 IS-IS术语

1）ES 终端系统

所有设备连入CLNP后都叫ES

TCP/IP CLNS/CLNP匹配

2）IS:ROUTER

3）CLNP 无链接网络协议 里面对应NET和NSAP

4）CLNS 无链接网络服务……类似于UDP

5）DIS 指定中间系统 类似DR

6：IS-IS中路由器的类型 router type

6.1）所有启用IS-IS的路由器，默认类型都是L1/L2：类似于ABR：将区域内路由器和骨干路由器连接

6.2）可改为L1：区域内路由器

6.2）可改为L2：骨干区域路由器

OSPF中骨干区域必须从0开始，IS-IS没有限制。

7：IS-IS neighbour TYPE 邻居关系类型

7.1）L1/L2 L1/L2——L1/L2（X）取交集

7.2）L2

7.3）L1

相同区域ID 不同区域ID

L1——L1/L2 取L1 L1——L1/L2 【L1是区域内路由器】（X）

【公务员考试找共同点？】

中国共产党和国民党……

L1——L1 L1——L1（X）

【L1设备只和相同区域设备建立邻居关系】

L1——L2（X） L1——L2（X）

不然L1/L2就没意义了呀。

L1设备只能和相同区域内带L1角色的设备形成L1邻居关系。【L1邻居关系——区域内邻居关系】

L1/L2——L1/L2 L1/L2——L1/L2

【此时既为L1也为L2】 【L1没有L2有】

L1是区域内邻居关系，L2首先是区域间邻居关系【区域号不一样】，但也可以是区域内邻居关系。

L1/L2——L2【L2】 L1/L2——L2【L2】

L2——L1（X） L2——L1（X）

8：IS-IS的特性

8.1）IS-IS中DIS的特点：类似于DR 指定路由器

！！回顾

OSPF中DR干嘛的？

①减少不必要的邻接关系；

②统筹更新发送过程。

图

EIGRP

OSPF DR BDR

1）DIS的选举

先比较优先级（默认是64，最大127）大则优

再比较sys-id 大则优

参数不同，机制相似。

2）没有B DIS，且DIS可以随意被替代。

3）所有非DIS之间，都可以正常形成邻接关系。

DIS用来干啥？原因：DIS被做了一半。

所以可以当它不存在了。没有任何作用。

这点上不如OSPF。

8.2）IS-IS中区域的特点

IS-IS也是层次化的设计，骨干层，常规层。同样也有骨干区域和常规区域

IS-IS不要求骨干区域编号一定是0

IS-IS不要求所有常规区域必须和骨干区域连接。

*IS-IS区域的边界是在路由器，OSPF区域边界是在接口

IS-IS允许接口和路由器在不同区域。

OSPF中拥有特殊区域：STUB T-STUB

NSSA、T-NSSA

IS-IS 区域内设备只要路由器角色改为L1，它所连接的L1/L2设备就自动给它一条缺省

并且不会将区域间路由，外部路由放入区域内。

计算公式及方法和OSPF一模一样，都叫COST

默认情况下所有接口都被赋予了10的开销【不是动态计算】

有计算公式但是不用

13：IS-IS 的重分发

外部网络进入isis，默认是L2路由类型，默认开销是10。

router isis

redistribute [rip/eigrp as-id] metric <指定外部路由初始度量值>

< level-1 /level-2/level-1-2 > //指定外部路由是什么类型进入isis

redistribute ospf pid metric-type < external ospf 的外部路由/internal ospf的区域内或区域间路由>

路由器角色为L1，那么重分发一般不允许做在这样的设备上。

如果L1角色设备需要配置重分发，那么一定要在最后加上level-1。

14：ISIS 的手动汇总

建议配置在L1/L2角色设备上。

汇总后路由类型为L2，开销默认10 。

L1设备上默认无法对本地 ISIS L1 路由进行汇总。

本地直连网络以外部路由 L1 形式进入，再针对 L1 路由汇总。

router isis

summary-address <汇总后网络> <汇总后网络掩码> level-1/level-2

15：ISIS 认证 及 ISIS 路由泄露

15.1）ISIS 认证

  1. 链路认证 isis password cisco

    该认证是利用 IIH 完成的，所以认证不同过影响邻接关系。链路认证是最高级别的，如果区域认证通过，链路认证不通过就是不通过。一般一个认证就行了。

  1. 区域认证 area-password cisco

    利用 LSP 完成的，认证不通过只影响路由的学习，不影响邻接关系。

  1. 域认证 domain-password cisco

    该认证如果不通过，只影响路由的学习，不影响邻接关系。

15.2）路由泄露

L1/L2 设备默认不允许 L2 数据库信息进入 L1 数据库，变成更新发送。

15.3）路由泄露的配置流程

步骤1：L1/L2 设备上编写 ACL，匹配被泄露的目标路由

步骤2：L1/L2 设备上编写 ROUTE-MAP 引用 ACL。

步骤3：L1/L2 设备上做 ISIS 到 ISIS 的重分发，并且指定是利用 route-map 完成 L2 数据库条目到 L1 数据库条目的泄露。

redistribute isis ip level-2 into level-1 route-map 2T1

更改开销 isis metric < 开销>
