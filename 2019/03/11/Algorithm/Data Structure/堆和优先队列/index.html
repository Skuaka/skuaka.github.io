堆和优先队列

概念

最小堆：所有父节点都比子节点小的完全二叉树。

最大堆同理。为了方便，本篇讲解都基于最小堆。

堆存储在数组内，相比于普通队列，寻找最大(小)值的速度非常快，所以堆也可以作为优先队列。

优先队列：支持插入元素和寻找最大(小)值元素的数据结构。

完全二叉树回忆：

从根节点编号为 1 开始从左往右编号：

若完全二叉树的父节点编号为 i，则左儿子为 2*i，右儿子为 2*i+1。

若完全二叉树的子节点编号为 i，则父节点编号为 i/2。

算法

向下调整

比如需要取出最小堆的根节点，并插入一个新值。

向上调整

在数组的最后插入一个新值，这时候就需要向上调整维持最小堆特性。

建堆

方法1：O(NlogN)

从空堆开始，依次向上往堆中插入每一个元素。

因为插入第i个元素所用的时间为O(logi)，所以整体是O(NlogN)。

方法2：O(N) 更好

直接把所有元素放入数组内，然后在这颗完全二叉树中，从最后一个非叶节点(n/2)开始，依次判断以这个节点为根的子树是否是最小堆。如果所有子树都是最小堆，那么整棵树就是最小堆。

时间复杂度的计算要用到级数公式，最终趋近于O(N)。

应用

堆排序

与快速排序一样，堆排序的时间复杂度也是 O(NlogN)。下面假设从小到大排序：

方法1：

先建立最小堆，然后每次删除顶部元素并输出顶部元素，然后将最后一个节点赋值到堆顶，然后向下调整。

方法2：（更好）

不建立最小堆而建立最大堆，每次把 h[1] 和 h[n] 交换，堆的大小减一，这样最大的元素就归位了。重复操作。

求第K大的数

题目意思就是：比如从1到10的整数，第2大的数就是9。

求解：先建立一个大小为K的最小堆，如果下一个元素比堆顶大，就把堆顶替换为这个元素，然后调整，最终堆顶就是第K大的数。

Dijkstra优化

Dijkstra算法中每次找离源点最近的一个顶点可以用堆来优化，使算法的时间复杂度降到 O( (M+N)logN )
