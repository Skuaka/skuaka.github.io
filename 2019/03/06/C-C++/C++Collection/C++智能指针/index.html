C++智能指针

C/C++
  • C/C++

为什么需要？

当程序正常或者异常终止时，普通指针在栈中被删除，但是它指向的内存不会被删除。

而如果这个指针是对象，则可以让它的析构函数删除指向的内存。

用法：将new获得（直接或间接）的地址赋给这种对象，当智能指针过期时，它的析构函数将使用delete来释放内存。当然，它们还有其他的功能。

比如：auto_ptr<string> ps(new string);

智能指针模版类

c++98 ：auto_ptr（废弃）

c++11 ：unique_ptr、shared_ptr

c++17 ：weak_ptr

boost库还有一些智能指针，本文不介绍。

将原程序换成智能指针的步骤

  1. #include <memory>，名称空间：std。
  2. 替换指针
  3. 删除delete语句

赋值的三种方法

如果将一个指针对象赋值给另一个，那么会 delete 同一个对象两次，所以产生了不同的智能指针。解决这个问题有下面三种方法：

深复制

定义赋值运算符，使之进行深复制，即两个指针指向不同的对象。

所有权模型

unique_ptr

auto_ptr和unique_ptr采用所有权模型，但是unique_ptr会在编译阶段就报错，所以已经取代了auto_ptr。

对象析构规则

The object is disposed of using the associated deleter when either of the following happens:

  • the managing unique_ptr object is destroyed
  • the managing unique_ptr object is assigned another pointer via operator=or reset().
赋值规则

程序试图将一个unique_ptr赋给另一个时，如果源unque_ptr将存在一段时间，编译器将禁止这样做，否则会造成多次使用delete。除了以下情况：

  • 如果源unique_ptr是一个临时右值，编译器允许这样做；

  • std::move( )可以将一个unique_ptr指针赋给另一个。它使用了C++11的移动构造函数和右值引用。

Notes

  • unique_ptr可以用于new[]，其他智能指针不行。

    经个人测试只能在构造函数内用，不能赋值，不能move。

  • Only non-const unique_ptr can transfer the ownership of the managed object to another unique_ptr. If an object’s lifetime is managed by a const std::unique_ptr, it is limited to the scope in which the pointer was created.

non-member function

引用计数模型

shared_ptr

shared_ptr在赋值时，该对象计数加一，指针过期时，计数减一；当最后一个指针过期时，才调用delete。

对象析构规则

The object is destroyed and its memory deallocated when either of the following happens:

  • the last remaining shared_ptr owning the object is destroyed;
  • the last remaining shared_ptr owning the object is assigned another pointer via operator= or reset().
STL和智能指针

很多STL算法都支持复制和赋值操作，这些操作可用于 shared_ptr，但不能用于 unique_ptr。

也可以将 unique_ptr 存储到 STL 容器中，只要不调用将一个 unique_ptr 复制或赋给另一个的方法或算法（如 sort ）。

unique赋给shared

  • shared_ptr 有一个用于将右值 unique_ptr 转换为 shared_ptr 的显示构造函数，然后接管对象。

  • 在 unique_ptr 为右值时，可以将其赋给 shared_ptr，这将一个 unique_ptr 赋给另一个需要满足的条件一样。

环形引用

如果两个类的智能指针成员互相指向了对方，导致自己的引用计数最少为1，就不会进行析构，这就造成了内存泄漏。

weak_ptr

为了避免shared_ptr的环形引用问题，需要引入一个弱引用weak_ptr， weak_ptr是为了配合shared_ptr而引入的一种智能指针，它更像是shared_ptr的一个助手而不是智能指针，因为它不具有普通指针的行为，没有重载operator* 和 ->，它的最大作用在于协助shared_ptr工作，像旁观者那样观测资源的使用情况.

weak_ptr本身也是一个模板类，但是不能直接用它来定义一个智能指针的对象，只能配合shared_ptr来使用，可以将shared_ptr的对象赋值给weak_ptr，并且这样并不会改变引用计数的值。

解决环形问题

为了解决上面例子中的环形引用问题，可以把当中的一个shared_ptr换成weak_ptr。

（虽然通过弱引用指针可以有效的解除循环引用，但这种方式必须在能预见会出现循环引用的情况下才能使用，即这个仅仅是一种编译期的解决方案，如果程序在运行过程中出现了循环引用，还是会造成内存泄漏的。因此，不要认为只要使用了智能指针便能杜绝内存泄漏。）

多线程

多线程读写 shared_ptr 要加锁

因为 shared_ptr 有两个数据成员，读写操作不能原子化。两个 shared_ptr 对象实体可以被两个线程同时写入，“析构”算写操作。

智能指针的实现

  1. 构造函数中计数初始化为1；

  2. 拷贝构造函数中计数值加1；

  3. 赋值运算符中，左边的对象引用计数减一，右边的对象引用计数加一；

  4. 析构函数中引用计数减一；

  5. 在赋值运算符和析构函数中，如果减一后为0，则调用delete释放对象。
