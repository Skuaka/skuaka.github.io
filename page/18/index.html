CCIE bgp

边界网关路由协议

BGP简介

工作类型：路径矢量路由选择协议

基于tcp(三次握手)

三张表：邻居表（peer表） bgp转发表 路由表

BGP版本

BGPv4：IPv4

BGPv4+：IPv6

MP-BGP：协议npls

M-BGP：多播

BGP优缺点

优点

1EGP路由表只能存储千条信息，bgp可上万条。

2丰富的路径属性，13条选路原则，提供了更丰富的路由操控策略。

缺点

1对路由器性能要求较高。

2配置程序比较复杂。

3一般小网络不建议使用bgp

AS 和 邻居类型

bgp AS 一台路由器只能属于一个

公网AS1-65411 65412-65535私有AS

iBGP：处于同一AS内的BGP路由器所建立的BGP对等体关系

eBGP：不同AS内的BGP路由器建立的…

###

BGP消息

open：建立成功，会互相发送open信息(类似hello)协商邻居关系的各项参数，等待对方确认。

keeplive：维持对等体(类似邻居)关系的建立，60s发一次，180s后peer关系结束

update：不会周期性更新，只会增量更新或触发更新。

Notification：当BGP检测到错误状态时，就向对等体发出Notification消息，之后BGP连接会立即中断。

peer关系建立过程

IDLE（空闲状态）：本地路由向外发送tcp(端口179)请求，等回复

connect状态：tcp建立

active：建立不成功，再次请求tcp连接的建立（回到IDLE）。

open sent：建立成功，发送open消息

openconfirm：收到对方的信息进行参数对比，参数一致则建立完成。

established：连接建立。。update。。keepalive。。

bgp的防环机制

iBGP-水平分隔原则

BGP对等体路由器收到iBGP路由，不会发送给其他iBGP对等体路由器。

但是纯水平拓扑上该原则会导致路由不被传递学习。两个解决方案

全互联拓扑解决方案：所有路由器间都建立iBGP关系。（垃圾方案）

路由反射器解决方案：

eBGP

基于AS PATH信息的检测。接收到来自于eBGP peer的路由更新，更新所带AS-PATH 信息包含了当前设备AS号，则该路由更新忽略。

特殊拓扑下路由反射器的防环机制

originate id：起源id。路由器将设备的RID设置为起源ID。起源ID和当前设备RID一样，则路由更新不学习。

cluster id：簇id。管理员在路由反射器上设置，路由更新携带的簇ID一致，则路由更新不学习。

同步与下一跳判断

路由更新来自于iBGP peer时才考虑同步和下一跳。

同步机制

对于当前启用BGP的设备，路由更新对应的网络底层协议也必须能学习到，这才叫同步。同步是为了避免路由黑洞。

现实中同步被关闭，路由黑洞问题无法避免，但利用MPLS能解决。

bgp路由黑洞

路由更新的传递设备也就是数据传输的沿途设备，并没有参与BGP ，学习该路由，所以数据抵达后，只能丢包。

解决方法：

1重分发(不建议，会增大路由表)

2全互联的网络拓扑(两两之间建立ibgp对等体)

下一跳判断

来自于iBGP peer的路由更新，如果下一跳不可达，路由更新不学习。

路由源于AS外，通告者是eBGP peer，下一跳就是eBGP peer地址。

路由源于AS外，通告者是iBGP peer，下一跳依旧是eBGP peer地址。此时下一跳可能不可达。

路由源于iBGP peer，通告者是iBGP peer，下一跳就是iBGP peer地址。

补充内容

BGP的路由更新属性

  1. 公认必选

    NEXT-HOP：下一跳

    AS-PATH：AS路径信息

    Originate code：起源代码

    ​ 描述了目标网络进入BGP的方式（I E ?）

    ​ I ：目标网络源于底层协议通过network进入BGP

    ​ E：EGP共享的目标网络信息

    ​ ?：外部路由以重分发的方式进入BGP

  2. 公认任选（自选）

    LP：本地优先

    ​ 路由更新在IBGP peer间传递时，LP值才出现，且为100

    ​ 作用：影响iBGP peer 出站路径的选择，LP参数越大越优。

    automatic aggregate：原子聚合

    ​ BGP发生了自动聚合或手动聚合时，聚合后网络携带该属性。

    ​ 原子聚合属性用于提醒更新接受者，路由更新目标网络可能丢失了部分原有属性。

  3. 任选可传递

    Aggregated：被聚合属性——聚合后的网络携带的属性，表明该网络是聚合来的。

    Community：团体属性 是携带特定策略的标记

    ​ 公认团体属性：

    ​ Internet：所有的BGP路由默认携带该属性，该属性对应的策略是，共享给所有的BGP peer。

    ​ no-export：不导出，携带该属性的路由更新不共享给eBGP peer。

    ​ no-advertise：不通告，携带该属性的路由更新不共享给任何BGP peer。

    ​ local-as：本地as，携带该属性的路由更新不共享给联盟内其他eBGP peer。

    ​ 自定义团体属性：

    ​ AA:NN

    ​ AS号：标记值

  4. 任选不可传递（属性保留，但值为空）

    MED：多出口距离

    ​ eBGP间路由共享携带该属性，MED值用于影响eBGP peer入站路径选择。MED值默认为0，越小越优。

    Originate id、cluster id

BGP选路原则

选路原则决定了BGP按什么标准进行路由更新属性的比较

  1. 优选拥有最高权重值的路径 weight

    用于影响当前路由器出站路径选择

    默认32768，越大越优

    不同类型的对等体出现，需要影响路径抉择，权重更优。

  2. 优选LP大的

  3. 优选起源于本地的路径

    next-hop=0.0.0.0

  4. 优选拥有最短AS-PATH 路径信息的

  5. 比较Origin属性<IGP优先于EGP优先于Incomplete>

  6. 优选拥有最佳MED值的

  7. eBGP peer 的路径优于 iBGP peer的路径

  8. 优选离下一跳更近的路径

  9. 针对eBGP peer的路由，通过命令行允许多条路径，成为最佳进入路由表

  10. 优选最早被学习的路由，只针对 eBGP peer

  11. 优选拥有最小RID的BGP peer提供的路由

  12. 优选拥有最短cluster-id 的路径

  13. 优选拥有最低peer地址的邻居提供的路径

BGP基本配置

对等体体必须手动指定，并且指定的地址可达

自动汇总 auto-summary

自动汇总用来设定BGP宣告是精确还是不精确的

auto-summary 关闭的情况下，BGP 处于精确宣告状态

auto-summary 开启的情况下，BGP 处于非精确宣告状态

被宣告的网络只要拥有本地对应的子网，被宣告网络就可以当做更新发送出去。

针对外部路由，auto-summary 开启，外部路由自动汇总，否则外部路由就明细进入。

路由更新属性

如何修改路由更新属性(三个步骤)

  1. 抓取被修改路由更新属性的目标网络

    • 利用ACL抓取
    • 前缀列表 prefix-list

  2. 利用路由映射调用 acl 或prefix-list 完成属性修改

  3. BGP里引用 route-map

weight 修改命令

通过 LP 影响出站路径

通过增加 AS-PATH 路径信息影响选路

只有 eBGP peer 间该属性才能修改，当前路由器只关心 AS-PATH 的长短，不关心是否作用于 eBGP或 iBGP

！！注意

AS-PATH 信息的附加 in 和out 是不一样的

1 i out方向附加666，就变成 1 666i

1 i in 方 向附加666，就变成 666 1

CCIE STP生成树协议

STP生成树协议 7.27

一个广播域中选举一个生成树。

广播风暴：绝不停止。

重复帧。

mac地址表的不稳定。

1一个网络中选取一个根桥

2根桥上所有端口都为指定端口

3非根桥上选取一个根端口

4每一段选举一个指定端口

5非指定端口为阻塞端口

*粗体为生成树定义的交换机接口角色

根端口：距离根桥距离最短的端口。开销值一样的话就选最小的发送者id。只在非桥根上选举，并且只选一个。

指定端口：可以执行数据的转发。一台交换机上可以有多个指定端口。根桥上所有端口都是指定端口。

阻塞端口：逻辑上的阻塞，不执行数据的转发，但会持续侦听bpdu信息。

BPDU(桥协议数据单元)只允许根桥产生

root id

优先级：(默认32768，越小越优，更改 优先值时必须为4096的倍数)。

背板mac：标识一台交换机，背板mac小的为优。

bridge id

优先级+背板mac：和根id一样

cost

开销值

port id

优先级(默认128，小优，更改时必须为64的倍数)

端口mac：发送者的端口mac

PVST：每vlan生成树，在优先级插入vlan的信息。优先级+vlan编号。

端口状态

Blocking（阻塞状态）：此时，二层端口为非指定端口，也不会参与数据帧的转发。该端口通过接收BPDU来判断根交换机的位置和根ID，以及在STP拓扑收敛结束之后，各交换机端口应该处于什么状态，在默认情况下，端口会在这种状态下停留20秒钟时间。

Listening（侦听状态）：生成树此时已经根据交换机所接收到的BPDU而判断出了这个端口应该参与数据帧的转发。于是交换机端口就将不再满足于接收BPDU，而同时也开始发送自己的BPDU，并以此通告邻接的交换机该端口会在活动拓扑中参与转发数据帧的工作。在默认情况下，该端口会在这种状态下停留15秒钟的时间。

Learning(学习状态)：这个二层端口准备参与数据帧的转发，并开始填写MAC表。在默认情况下，端口会在这种状态下停留15秒钟时间。

Forwarding（转发状态）：这个二层端口已经成为了活动拓扑的一个组成部分，它会转发数据帧，并同时收发BPDU。

Disabled（禁用状态）：这个二层端口不会参与生成树，也不会转发。

交换机指示灯：

绿色常亮：接口两端没有问题，但是当前没有数据从中通过

\6. 1025–4094 扩展 仅仅用于以太网VLAN

多播（multicast）

基础知识

数据的传递方式

  • 单播（unicast）

    一对一通讯，数据接收者已知。

  • 广播（boradcast）

    一对所有通讯，数据接收者未知。

  • 多播（mulicast）

    一对多通讯，数据接收者未知。

路由器和交换机对不同类型数据的处理方式

路由器

单播包：路由或丢弃

广播包：丢弃或本地存在感兴趣的进程进行处理

多播包：丢弃或本地存在感兴趣的进程进行处理

交换机

单播帧：目的MAC地址交换机是否已知，如果已知则交换，如果未知则泛洪

广播帧：泛洪

多播帧：泛洪

IGMP 互联网组管理协议

路由器及终端运行，帮助路由器发现直连子网中的多播接收者。

组（group）：所有对同一多播流感兴趣的成员组成了多播组。

组成员（group member）：组中的多播流接收者。

版本：IGMPv1、IGMPv2、IGMPv3

IGMPv1

  • 报告消息（report）：始发于终端，用于通告子网中的路由器存在特定组组成员。

    发送终端的IP 和MAC以及组信息。

  • 查询消息（request）：始发于路由器，用于查询直连子网中是否存在组成员。

    通常是路由器接收到多播流后触发。

一旦发现组成员，路由器就创造一个180s 的老化计时器，该计时器通过 report 消息进行维护。

IGMPv1的缺陷

  1. 响应时间慢

    组中没有成员时，路由器需要 180s 才能发现

  2. 存在多查询者

  3. 存在多转发者

IGMPv2

  • 报告消息（report）：和 v1 相同

  • 查询消息（request）：和 v1 相同

  • 离开消息（leave）：始发于终端，终端PC 不再是组成员时发送，用于通告路由器有组成员离开。

    收到离开消息时，立刻发送SQ，如下。

  • 特定查询消息（SQ）：始发于路由器，发送出去后设定的老化时间为 3s 。如果 3s 内收到 report ，说明发送离开消息的组成员并不是组中最后一个人；如果收不到任何消息，那么说明组成员不存在。

IGMPv2 查询者竞选机制：多个查询者通过比较查询消息源 ip 来确认谁最终负责查询，源 ip 越小越优。

多转发者缺陷 IGMPv2 无法克服。

多播地址

多播 IP 和多播 MAC 都不是用来定位接收者所在的网络和物理位置。

它们用于标识服务和应用。

RIPv2 224.0.0.9

IPv4 D类 224.x.x.x 239.x.x.x

多播 mac

设备根据多播的 IP 自动计算而来。

多播的MAC地址可能会重复，多播的 IP 后 23bit 一致则会重复。

多播路由器协议

是运行在路由器之间，负责建立多播路由路径，用于传递多播数据。

多播路由协议分类

  • 距离矢量：DVMRP 距离矢量多播路由协议（淘汰了）
  • 链路状态：MOSPF 多播的OSPF（非主流）
  • 距离向量：CBT 基于核心的树（非主流）
  • 独立于其他：PIM 协议独立多播（Cisco、主流）

PIM 的两个版本

PIMv1：用的是 IGMPv1，但是现在也用 IGMPv2

PIMv2：用的是 IGMPv2

PIM 的三种模式

  • 密集模式（dense mode）

    设备间带宽资源紧张，子网成员大部分是组成员的时候，建议使用密集模式。

    密集模式默认采用 IGMPv1。

  • 稀疏模式（sparse mode）

    设备间带宽资源依旧紧张，子网成员只有少数是组成员，建议使用稀疏模式。

    稀疏模式采用 IGMPv2。

  • 混合模式（sparse-dense mode）

    该模式下路由器先启用稀疏模式，稀疏模式失效后再用密集模式。

PIM 的密集模式详解

消息

  • 加入消息（join）：下游路由器通过加入消息将直连网络加入多播转发路径。
  • 修剪消息（prune）：下游路由器通过修剪消息，将原本隶属于多播路径的直连网络从多播路径中删除。
  • 接枝消息（graft）：下游路由器通过该消息请求将原本被拆除的多播路径再恢复过来。
  • 接枝确认消息（graft-ack）：上游路由器通过该消息告诉下游，直连路径恢复成多播转发路径。
  • assert消息（assert）：连接子网的路由器通过该消息，竞选出最终的转发者。

密集模式工作过程

扩散、剪枝。

PIM 的特殊机制

  • PIM 密集模式的 DR

    接口优先级+接口ip地址，构成竞选参数，越大越优。

    PIM密集模式早期使用IGMPv1，IGMPv1会出现多查询者问题，于是就利用DR的竞选，来帮助IGMPv1确认谁负责查询。

    PIM密集模式现在使用的是IGMPv2，DR就没用了。

  • PIM 消息中的 assert 作用

    通过竞选选出一个人负责转发数据。

    assert 消息，携带了三个竞选参数：

    • 当前路由器启用的底层协议管理距离，越小越优。
    • 当前路由器抵达多播源底层路由协议的开销，越小越优。
    • 当前路由器连接子网的接口 ip，越大越优。

PIM 密集模式的配置流程

  • 步骤1：确认参与多播的设备及接口

    所有需要接收多播流的接口和发送多播流的接口及连接组成员的接口，都要参与多播路由协议的启用。

  • 步骤2：路由器接口模拟成多播组成员

    注意：该命令可以反复录入，一个接口扮演多个组成员

  • 步骤3：路由器开启多播路由功能，接口启用多播密集模式

  • 步骤4：校验查看

    校验路由器是否发现了组成员：show ip igmp group

    校验路由器是否形成了PIM 邻居关系：show ip pim neighbor

    校验路由器多播路由表：show ip mroute

多播路由表

父条目与子条目

  • 父条目

    父条目起提醒注释的作用，告知当前路由器和管理员，路由器可能需要为某个多播组传递流量。

    路由器收到 report 消息便会产生父条目，路由器收到多播流也会产生父条目。

  • 子条目

    子条目是实际负责多播流量传递工作。

    子条目是路由器根据计算得来。

多播路由条目的样式

多播防环机制

RPF：反向路径转发：启用多播的接口默认开启，用于避免环路的出现。

所有接收多播流的接口，必须通过 RPF 检测，才可以成为有效的上游接口。

当前接口必须是抵达多播源下一跳地址的本地出口，才通过检测。

用来查看rpf检测是否通过：show ip rpf <intf-id>

多播树

多播树其实就是多播转发数据的路径。

密集模式的树

最短路径树，也称源树。

当前路由器以多播源为根，计算抵达多播源的最短路径，每个路径对应到多播路由表中的一个条目。

源树缺陷
  • 容易产生大量的多播子条目。
  • 树构建的早期多播流被扩散到了不必要的设备上

PIM 的稀疏模式详解

消息

join、prune、graft、graft-ack、assert（和密集模式相同）

注册消息（register）

注册停止消息（register-stop）

稀疏模式中的集合点：RP

RP 是管理员指定的，是多股多播流的交汇处。

稀疏模式的工作原理

上游注册，下游加入，RP 整合。

！！注意

RP 以下的设备，以 RP 为根，计算最短路径。

RP 自身以源为根，计算最短路径。

稀疏模式是一个双树结构，RP 和源之间维护的是源树，下游设备与 RP 之间维护的是共享树。

共享树切换成源树

稀疏模式下，每台支持多播路由协议的路由器在接收超过4个字节的多播流后，就会尝试建立源树。

      RP

    查看组地址、本地接口、报告者 是否出现

    错点

    • 没有模拟成组成员，缺乏接口下：ip igmp join-group <a.b.c.d>
    • 路由器接口没有加入PIM，缺乏接口下:ip pim <x>
    • 连通性问题

  2. 确认稀疏模式获取 RP 信息

    show ip pim rp mapping 查看 RP 映射，查看 RP 的结果

    show ip pim rp 查看多播组和 RP 的关联

    错点

    • 没有通过静态 RP 指明 RP 地址
    • BSR 的 C-BSR 或 C-RP 配置问题
    • auto-rp 的配置问题
    • 缺乏auto-rp 侦听的配置
    • 设备间缺乏必要的PIM 邻居：show ip pim neighbor
      • 设备缺乏多播路由功能和接口忘记加入PIM
      • 接口忘记激活、错误的安全策略挡住了PIM消息
    • 设备无法正常抵达 RP 地址，所以不显示获取的 RP 消息
      • 路由层面问题、忘记宣告、忘记重分发、错误的过滤

  3. RPF 检测未通过

    • 缺乏必要的更新源路由
    • 忘记宣告、忘记重分发、错误的路由更新过滤

  4. 是否是跨域多播、缺乏必要的配置和连通性

    • 缺乏必要的 MSDP 配置和 MSDP 对等体关系

      show run | sec ip msdp

      show ip msdp summary

    • RPF 检测不通过

CCIE 重分发

重分发

将不同路由协议转化

1直连进路由选择协议

redistribute connect metric 1

2其他路由选择协议进rip

router rip

router下 redistribute eigrp 100 metric 1值(默认0)

3其他路由选择协议进eigrp

router eigrp 100

redistribute rip metric 10000带宽 100延迟 255可靠性 1负载 1500MTU值（默认为以上数字，一般不改）

4其他进ospf

router os 1

redistribute eigrp 100进程号 subnets（子集，不加则为主类网络）

默认OE2

将oe2改成oe1:

re ei 100 subnets metric-type 1 (+metric 值）

重分发流量控制

acl

奇acc 1 permit 172.16.1.0 0.0.254.255

偶acc 1 permit 172.16.0.0 0.0.254.255

将acl抓去到的流量给route-map

route-map +名字 +permit/deny

优先级10(默认优先级)

match ip address +acl编号

route-map也默认deny所有流量

如需放行所有流量 route-map 名字 permit +下一个优先级20(一般为10的叠加)

re rip router-map ccie metric ……
